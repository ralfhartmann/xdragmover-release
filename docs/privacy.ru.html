Конфиденциальность

Хостинг сайта

Этот сайт размещён на GitHub Pages. Поставщик услуги — GitHub, Inc., 88 Colin P. Kelly Jr. Street, San Francisco, CA 94107, США, либо GitHub Ireland Limited, 70 Sir John Rogerson's Quay, Dublin 2, Ирландия.

При посещении этого сайта GitHub получает и обрабатывает технические данные, в частности IP-адрес, тип браузера, операционную систему, referrer-URL, дату и время обращения, а также запрошенные файлы. Эти данные сохраняются в серверных логах и используются для предоставления, безопасности и стабильности сайта. Подробности см. в заявлении о конфиденциальности GitHub.

DNS и Cloudflare

Серверы имён (nameservers) этого домена полностью делегированы Cloudflare. В рамках предоставления DNS Cloudflare обрабатывает персональные данные, в частности IP-адреса и DNS-запросы, технически необходимые для разрешения домена и доступности сайта. Поставщик — Cloudflare, Inc., 101 Townsend St., San Francisco, CA 94107, США, а также Cloudflare Germany GmbH, Rosental 7, 80331 Мюнхен, Германия.

Передача данных в США осуществляется на основании EU-US Data Privacy Framework и дополнительных стандартных договорных положений. Подробности см. в политике конфиденциальности Cloudflare.

Никаких cookie, никакого отслеживания

Сам этот сайт не устанавливает cookie и не содержит собственных скриптов аналитики, рекламы или отслеживания сторонних сервисов. Технически необходимая обработка для доставки страницы, такая как разрешение DNS или серверные логи, сюда не относится и описана выше.

Само приложение

XDragMover не собирает, не передаёт и не хранит никаких персональных данных. Единственная сетевая активность — это опциональная, по умолчанию отключённая проверка обновлений, которая обращается к публичному репозиторию релизов этого проекта на GitHub — подробности см. в README самого приложения.

Правовые основания и сроки хранения

Обработка технических данных для предоставления сайта осуществляется на основании ст. 6 (1) (f) GDPR (законный интерес: эксплуатация, безопасность и стабильность сайта). Данные, которые вы бы передали активно, обрабатывались бы на основании ст. 6 (1) (a) GDPR (согласие) или (b) (преддоговорные меры), в зависимости от цели.

Данные логов обычно хранятся лишь короткое время. GitHub и Cloudflare могут хранить определённые логи разное время; как правило, операционные логи и логи безопасности удаляются в течение нескольких часов — дней. Точные сроки регулируются политиками конфиденциальности соответствующих поставщиков.

Ваши права

Вы имеете право на доступ, исправление, удаление, ограничение обработки, возражение, а также на переносимость данных. Для реализации своих прав или по вопросам конфиденциальности, пожалуйста, свяжитесь со мной по адресу электронной почты, указанному в разделе Выходные данные.

Контакт и ответственный за обработку данных

Ответственным за обработку данных в смысле GDPR является лицо или организация, указанные в разделе Выходные данные. Контактные данные приведены там же.

Дополнительные сведения для международных посетителей

Разделы выше уже исчерпывающе описывают, какие данные обрабатываются — а именно исключительно технические данные доступа от GitHub Pages и Cloudflare, без cookie, без отслеживания и без продажи или передачи данных третьим лицам. Ниже это дополнительно поясняется для посетителей из отдельных юрисдикций.

Европейский союз (GDPR). Для посетителей из ЕС описанный выше GDPR уже применяется в полном объёме — цель, правовое основание, срок хранения и ваши права описаны в разделах «Правовые основания и сроки хранения» и «Ваши права» выше.

Калифорния (CCPA/CPRA). Этот сайт не продаёт и не «передаёт» (в смысле CCPA/CPRA) никакие персональные данные и не создаёт профили пользователей. Ссылка «Do Not Sell or Share My Personal Information» поэтому не требуется, так как нечего продавать или передавать. Посетители из Калифорнии тем не менее имеют право на получение информации, удаление данных и защиту от дискриминации; запросы направляйте по контактному адресу, указанному в разделе «Выходные данные».

Бразилия (LGPD). Поскольку этот сайт не собирает и не обрабатывает персональные данные сверх технически необходимых, отдельный ответственный за защиту данных (DPO) по LGPD не назначен. Ответственным в смысле LGPD является лицо, указанное в разделе «Выходные данные»; субъекты данных в Бразилии могут реализовать свои права по указанному там контактному адресу.

Китай (PIPL). Этот сайт не ориентирован специально на посетителей из Китая и не экспортирует персональные данные из Китая. Если доступ всё же осуществляется из Китая, обработка ограничивается описанными выше техническими данными доступа через GitHub Pages и Cloudflare; отдельное уведомление о трансграничной передаче данных по PIPL не требуется ввиду отсутствия активного сбора данных.

Индия (DPDP). Этот сайт не запрашивает согласие на обработку персональных данных и поэтому не нуждается в отдельном многоязычном уведомлении согласно индийскому Закону о защите цифровых персональных данных (DPDP). Описанные выше технические данные доступа — единственная обработка, происходящая также при доступе из Индии.
